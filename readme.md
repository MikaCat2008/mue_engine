# MUE Engine

## Описание

Забудьте о скучных фреймворках. MUE Engine - это мощный игровой движок, который рвёт шаблоны и строит будущее 2D-игр. Основанный на чистой архитектуре Entity-Component-System (ECS), он даёт вам свободу, о которой вы мечтали. С нашим собственным XML-подобным языком разметки ECSL (Entity Component System Language) вы сможете декларативно описывать игровые сущности, компоненты и системы - просто, элегантно, без лишней суеты.

Этот проект - не просто код. Это страсть к созданию чего-то великого. В активной разработке, с базовыми возможностями для сцен, персонажей, тайлов и UI. И это только начало.

## Особенности

- **ECS Архитектура**: Чистая, как слеза младенца. Гибкая система, которая позволяет управлять объектами так, как вам удобно. Никаких компромиссов.
- **ECSL Язык Разметки**: Наш собственный язык. Описывайте сцены в файлах `.ecsl` - просто, мощно, без боли.
- **Pygame-CE**: Рендеринг и ввод на уровне. Pygame-ce делает графику плавной, а управление - отзывчивым.
- **Модульная Структура**: Разделён на модули: core (ядро, которое держит всё вместе), canvas (рендеринг с умными слоями), base (системы, которые работают на вас).
- **Поддержка Тайлов и Структур**: Встроенная система для тайлов и объектов. Строите миры? Легко.
- **Инвентарь и UI**: Базовые компоненты для интерфейса. Всё под контролем.

## Архитектура

### Entity-Component-System (ECS)

Это сердце движка. Забудьте о монолитах.

- **Entity**: Ваши игровые объекты. Содержат компоненты и дочерние сущности. Всё организовано.
- **Component**: Данные и свойства. Позиция, текстура, идентификатор - что угодно. Гибкость на максимум.
- **System**: Логика, которая оживает. Рендеринг, управление игроком, ресурсы. Обновляет мир каждый кадр.

### ECSL

ECSL - наш секретный соус. XML-подобный язык для иерархии сущностей. Не код - магия. Пример:

```xml
<body>
    <Canvas>
        <CanvasLayer>
            <Player>
                <components>
                    <Identity>
                        <property name="tag" value="main-player" />
                    </Identity>
                </components>
            </Player>
        </CanvasLayer>
    </Canvas>
</body>
```

Это просто. Это мощно. Это будущее.

## Установка

Не тратьте время зря.

1. Python 3.8+ - у вас есть? Отлично.
2. Зависимости:
   ```
   pip install pygame-ce
   ```
3. Клонируйте или скачайте. Готово.

## Запуск

Запустите и почувствуйте силу:
```
python main.py
```

PygameExecutor берёт управление. Полноэкранный режим, 512x288 - идеально для старта.

## Структура Проекта

```
mue_engine/
├── main.py                 # Точка входа. Отсюда начинается приключение.
├── pygame_executor.py      # Исполнитель на Pygame. Рендерит мир.
├── body.ecsl               # Основная сцена. Ваш холст.
├── info.txt                # Информация о разработке. Планы и победы.
├── assets/                 # Ресурсы. Текстуры, спрайты - душа игры.
│   ├── entity/             # Спрайты сущностей. Герои оживают.
│   ├── structures/         # Спрайты структур. Мир наполняется.
│   ├── tile/               # Тайлы. Основа уровней.
│   └── ui/                 # UI элементы. Интерфейс под контролем.
├── engine/                 # Движок. Сердце системы.
│   ├── core/               # Ядро ECS. Entity, Component, System.
│   ├── base/               # Базовые компоненты и системы. Фундамент.
│   ├── canvas/             # Рендеринг с слоями. Видите мир.
│   └── executor.py         # Базовый исполнитель. Запускает всё.
└── ecsl/                   # Парсер ECSL. Превращает разметку в реальность.
```
